# Documentation Authorship

*Authorship: Agent-written.*

This repo uses visible authorship labels on documentation so the reader can tell what kind of voice they are reading.

## Labels

- `User-written` — written directly by the project author
- `Gently user-edited` — rooted in direct author wording and voice, with light structural editing, spelling correction, and only minimal summarization
- `Agent-written` — written primarily by an AI agent for mechanical, operational, or support clarity

## Intended Use

- install, command, support, and safety docs can be `Agent-written`
- principle, intent, and product-voice documents should prefer `User-written` or `Gently user-edited`
- changelog and roadmap entries can be agent-written as long as they stay concrete and honest

Code in this repo should be assumed AI-generated by default unless stated otherwise.
